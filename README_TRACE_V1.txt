Prasia Guild Trace v1 적용 메모

적용 기준:
- 기본은 기존 92레벨 이상 지문을 유지합니다.
- 92레벨 이상 관측 레벨이 없는 결사만, 해당 결사의 실제 존재 상위 3개 레벨을 기준으로 match_* 지문을 생성합니다.
- 예: 93/92/91 존재 → 92+ 기준, 91/90/89 존재 → 91/90/89 기준, 89/88/86 존재 → 89/88/86 기준.

변경 파일:
- scripts/collect_snapshot.py
  - 새 스냅샷 생성 시 match_rule, match_levels, match_*_distribution 필드를 추가합니다.
- js/compare.js
  - 웹 비교 시 match_* 지문을 우선 사용합니다.
  - 기존 92+ 필드만 있는 과거 스냅샷도 가능한 범위에서 보정해 비교합니다.
- index.html, js/app.js, css/style.css
  - 안내 문구를 Trace v1 기준으로 변경했습니다.
  - 결과 필터에 도착 서버 선택 필터를 추가했습니다.
- data/snapshots/2026-06-25_1150/*
  - 0625_0611.zip 기준 서버 이전 전 스냅샷을 Trace v1 양식으로 교체했습니다.

사용 방법:
1. GitHub에 전체 파일을 덮어씌웁니다.
2. Actions에서 Update Snapshot을 수동 실행하거나 자동 실행을 기다립니다.
3. 웹에서 이전/이후 스냅샷을 선택 후 비교합니다.
4. 결과 화면에서 '전체 서버' 또는 특정 도착 서버를 선택해 몰린 서버 후보만 볼 수 있습니다.
